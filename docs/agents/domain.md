# 领域文档

说明工程技能在浏览代码库时如何消费本仓库的领域文档。

## 探索前请先阅读

- **`CONTEXT.md`**（仓库根目录），或
- **`CONTEXT-MAP.md`**（仓库根目录，如果存在的话）——它指向每个上下文对应的 `CONTEXT.md`。阅读与你即将处理的主题相关的每个上下文文件。
- **`docs/adr/`** —— 阅读与你即将工作的领域相关的 ADR。在多上下文仓库中，也请检查 `src/<context>/docs/adr/` 下的上下文相关的决策记录。

如果这些文件不存在，**直接跳过**。不要标记它们的缺失，也不要建议立即创建。`/domain-modeling` 技能会在术语或决策真正被确定时惰性地创建它们。

## 文件结构

单上下文仓库：

```
/
├── CONTEXT.md
├── docs/adr/
│   └── ...
└── src/
```

## 使用术语表中的词汇

当你的输出中提到某个领域概念时（在 Issue 标题、重构建议、假设、测试名称中），使用 `CONTEXT.md` 中定义的术语。不要偏离到术语表明确避免的同义词。

如果所需的术语尚未出现在术语表中，这是一个信号——要么你在发明项目不使用的语言（重新考虑），要么确实存在空缺（记录下来给 `/domain-modeling`）。

## 标记 ADR 冲突

如果你的输出与现有 ADR 相矛盾，请明确指出，而不是静默覆盖。
